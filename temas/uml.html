🎯 UML: El Lenguaje Universal del Diseño de Software
¿Qué es UML?
UML es como los planos de un arquitecto, pero para software. Nos permite visualizar, especificar y documentar sistemas antes de construirlos, ahorrando tiempo y reduciendo errores.
¿Por qué es importante?
Facilita la comunicación entre equipos, reduce malentendidos, permite detectar problemas temprano y sirve como documentación del sistema.

💡 Analogía Práctica

Imagina que quieres construir una casa. ¿Empezarías directamente con los ladrillos? No, primero harías planos arquitectónicos. UML son los "planos" del software que nos ayudan a visualizar la estructura antes de programar.

🔗 Tipos de Relaciones UML

Dependencia
A - - -> B
"A usa B temporalmente"
Asociación
A ──── B
"A se relaciona con B"
Agregación
A ◇──── B
"A tiene B (débil)"
Composición
A ◆──── B
"A contiene B (fuerte)"
Herencia
A △
   │
   B
"B es un tipo de A"
Realización
A △
   ┊
   B

📝 Herramientas de Captura de Requisitos
🎭 Casos de Uso
Describen cómo los usuarios interactúan con el sistema para lograr objetivos específicos. Son detallados y formales, ideales para documentación completa.
📖 Historias de Usuario
Descripciones breves y simples de funcionalidades desde la perspectiva del usuario. Son ágiles y flexibles, perfectas para desarrollo iterativo.

⚖️ Comparación: Casos de Uso vs Historias de Usuario

📋 Casos de Uso
Ventajas: Muy detallados, documentación completa, incluyen flujos alternativos
Desventajas: Pueden ser rígidos, requieren más tiempo
Mejor para: Proyectos grandes, equipos distribuidos, documentación formal
📝 Historias de Usuario
Ventajas: Ágiles, flexibles, centradas en el usuario, fáciles de entender
Desventajas: Pueden ser ambiguas, requieren comunicación constante
Mejor para: Desarrollo ágil, equipos pequeños, iteraciones rápidas
